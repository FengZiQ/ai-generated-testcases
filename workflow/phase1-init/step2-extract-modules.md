# 阶段一·第二步：提取产品模块结构

## 目标

从需求文档中解析出产品的模块层级结构和每个模块的功能点。

## 处理流程

1. 按文档章节/标题识别模块边界
2. **识别产品所属行业**：
   - 从需求文档判断行业类型（金融/医疗/电商/教育/社交/SaaS/物流/出行/其他）
   - 判断业务地域（中国大陆/欧美/跨境/全球）
   - 判断目标用户类型（C端/B端/混合）
3. 对每个模块：
   - 识别子功能
   - 提取功能描述
   - 识别输入条件和预期行为
   - 识别异常处理描述
4. 识别模块间的依赖关系
5. 识别外部系统集成点

## 功能点提取规则

- 每个独立的可测试功能点记录一条
- 功能点包含：描述、输入条件、预期行为、异常处理
- 如果需求描述模糊，记录假设
- 不做测试设计，只做功能提取

## 输出

产品模块结构表：

```
### MOD-001：[模块名]
**功能点清单：**
- FP-MOD001-001：[功能点描述]
  - 输入条件：[描述]
  - 预期行为：[描述]
  - 异常处理：[描述]
- FP-MOD001-002：[功能点描述]
  ...

模块依赖：[依赖的其他模块]
```

行业信息（独立输出）：

```
## 行业信息
- 行业分类：[金融/医疗/电商/教育/社交/SaaS/物流/出行/其他]
- 业务地域：[中国大陆/欧美/跨境/全球]
- 目标用户类型：[C端/B端/混合]
- 初步判断适用法规：[法规列表（参考 reference/industry-regulations.md）]
- 涉及用户数据处理：[是/否] — 如涉及，列出数据类型
- 涉及资金/交易处理：[是/否]
- 涉及未成年人：[是/否]
- 涉及用户生成内容：[是/否]
```
