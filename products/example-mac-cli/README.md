# Mac CLI Toolkit — 参照示例

## 这个示例是做什么的？

本目录是一个**完整的参照示例**，展示使用本框架对一个产品进行测试的全流程输出。

## 你可以从这个示例中学到什么？

### Phase 1：产品初始化（`product-overview.md` + `modules/`）

当你有一个新产品需要进行测试时，框架会输出什么。

| 文件 | 说明 |
|------|------|
| `product-overview.md` | 产品初始化报告：产品概述、行业分类、适用法规清单、模块结构、功能点清单、风险评估 |
| `modules/module-*.md` | 各模块详情：功能点列表（ID、输入条件、预期行为、异常处理） |
| `modules-index.md` | 模块索引与依赖关系图 |
| `product-habits.md` | 产品级记忆：缩写约定、字段行为模式、历史积累的知识 |

### Phase 2：迭代测试（`iterations/v1.0/test-points.md`）

当产品有新迭代需求时，框架会输出什么。

| 文件 | 说明 |
|------|------|
| `test-points.md` | 完整迭代测试计划：变更分析→影响范围→测试点（功能/SEC/PERF/CMP）→探索式测试→思维导图→追溯矩阵→统计 |

## 如何使用？

**作为参考**：阅读各文件，了解框架输出的格式和质量标准。

**作为模板**：将你的产品名替换"Mac CLI Toolkit"，按相同结构组织你的产品目录。

## 产品简介

**Mac CLI Toolkit** 是一个虚构的 macOS 命令行系统管理工具集，包含文件管理、进程管理、网络诊断、系统信息四个模块。选择一个 macOS 用户熟悉的 CLI 产品作为示例，是因为它：
- 人人都能理解，降低学习成本
- 命令行产品的边界清晰，便于展示测试技术
- 涉及文件系统、进程、网络、系统信息等多维度，适合展示 SEC/PERF/CMP

> 💡 本目录中的"需求文档"为虚构，仅用于演示框架输出格式。
