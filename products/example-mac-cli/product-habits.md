## 模块缩写约定

| 模块 | 缩写 | 说明 |
|------|------|------|
| 文件管理 | FILE | File Management |
| 进程管理 | PROC | Process Management |
| 网络诊断 | NET | Network Diagnostics |
| 系统信息 | SYS | System Information |

## 命令行产品的共性规律

### 参数命名风格
- 短选项使用单字母：`-r` `-f` `-i`
- 长选项使用双横线：`--recursive` `--force` `--interactive`
- 选项值格式：`--count 4`（空格分隔）或 `--count=4`（等号分隔）
- 布尔选项：存在即为 true，无 `--no-` 前缀

### 路径参数行为
- 相对路径：相对于当前工作目录解析
- 绝对路径：以 `/` 开头
- 通配符扩展：由 shell 负责，工具收到的是展开后的列表
- 路径中包含空格：需用户自行引号包裹

### 权限模型
- 工具自身权限 = 当前用户权限
- 无法提升权限（sudo 需用户在外部使用）
- 文件操作受 macOS 文件系统权限约束
- 系统信息读取受 SIP（System Integrity Protection）约束

### 错误处理惯例
- 返回值 0 = 成功
- 返回值 1 = 一般错误
- 返回值 ≥2 = 特定错误类型
- 错误信息输出到 stderr
- 正常输出到 stdout

### 输出格式
- 默认：纯文本表格/列表
- 支持 `--json` 输出（各模块通用规划）
- 支持 `--quiet` 静默模式（仅返回状态码）

## 测试注意事项
- CLI 工具的边界测试重点：参数解析、路径处理、权限场景
- 网络模块测试需注意：不能在无网络环境执行实际网络调用
- 进程管理测试需注意：不要操作系统关键进程
- 系统信息测试需注意：不同 macOS 版本的输出可能不同
