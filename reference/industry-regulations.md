# 行业规则与法律法规参考

本文件列出各主要行业的常见法规和合规要求，供生成测试点时参考。
**注意**：此列表仅为常见法规摘要，不构成法律建议。实际合规测试应咨询法务团队。

---

## 金融/银行/支付

### 中国法规
| 法规名称 | 关键要求 | 影响的功能点 |
|---------|---------|-------------|
| 《网络安全法》 | 数据加密存储、用户信息保护、安全等级保护 | 登录、用户数据、日志 |
| 《个人信息保护法》 | 明示同意、最小必要、数据本地化 | 注册、隐私设置、数据导出 |
| 《数据安全法》 | 数据分类分级、重要数据出境评估 | 数据管理、数据传输 |
| 《非银行支付机构网络支付业务管理办法》 | 支付限额（Ⅰ类1000元/终身、Ⅱ类10万/年、Ⅲ类20万/年） | 支付、账户等级 |
| 《金融消费者权益保护实施办法》 | 信息披露、知情权、自主选择权 | 费率展示、账单、通知 |
| 反洗钱规定 | 大额交易报告（5万元人民币/等值外币）、可疑交易监测 | 交易、风控 |
| PCI DSS（支付卡行业数据安全标准，跨境） | 信用卡号加密、存储限制、定期安全扫描 | 支付、卡信息处理 |

### 通用
| 要求 | 说明 | 测试关注点 |
|------|------|-----------|
| SOX（萨班斯法案，赴美上市企业） | 财务报告内部控制 | 审计日志、权限控制 |
|  Basel III（巴塞尔协议III） | 资本充足率、流动性覆盖率 | 风控模型、压力测试 |
|  GDPR（服务欧盟用户） | 数据主体权利、72小时泄露通知 | 用户数据、隐私、删除功能 |

### 金融行业特有测试关注点
- 利息/费率计算的精度（保留几位小数？四舍五入规则？）
- 交易日志的审计追溯性（谁、什么时间、做了什么、IP等）
- 资金安全的双向认证/复核机制
- 账务日切/对账机制
- 超时/挂账/冲正等异常交易处理

## 医疗健康

### 中国法规
| 法规名称 | 关键要求 | 测试关注点 |
|---------|---------|-----------|
| 《医疗机构病历管理规定》 | 病历保存≥15年，门诊病历≥15年 | 数据留存、归档 |
| 《互联网诊疗管理办法（试行）》 | 线上诊疗需实体医疗机构支撑 | 医生资质、预约流程 |
| 《健康医疗大数据标准、安全和服务管理办法》 | 数据脱敏、授权访问 | 数据展示、权限 |
| 《药品管理法》 | 处方药销售管理 | 药品展示、购买流程 |

### 国际
| 法规 | 关键要求 | 说�� |
|------|---------|------|
| HIPAA（美国） | PHI保护、隐私规则、安全规则 | 患者健康信息加密、访问控制、审计 |
| FDA 21 CFR Part 11（美国） | 电子记录/签名的等效性 | 电子签名、审计追踪 |

### 医疗行业特有测试关注点
- 患者数据的严格权限控制（谁可以看哪些数据）
- 用药剂量计算（小数点、单位转换）
- 危急值提醒机制
- 电子病历的不可篡改性（审计日志）
- 系统可用性（医疗场景下系统宕机的严重后果）

## 电子商务/零售

### 中国法规
| 法规名称 | 关键要求 | 测试关注点 |
|---------|---------|-------------|
| 《电子商务法》 | 经营者登记、商品服务信息披露、交易记录保存≥3年 | 商家入驻、商品展示、订单 |
| 《网络交易监督管理办法》 | 禁止刷单炒信、明示优惠规则 | 评价系统、促销活动 |
| 《消费者权益保护法》 | 7天无理由退货、三包政策 | 退货流程、售后 |
| 《广告法》 | 禁止虚假宣传、极限词使用限制 | 商品描述、广告位 |
| 《价格法》 | 明码标价、禁止价格欺诈 | 价格展示、优惠计算 |

### 通用
| 要求 | 说明 |
|------|------|
| 儿童在线隐私保护法(COPPA) | 面向儿童的产品需家长同意 |
|  ADA/WCAG 无障碍访问 | 屏幕阅读器支持、色彩对比度 |

### 电商行业特有测试关注点
- 价格计算准确（商品价×数量+运费-优惠+税）
- 库存超卖（高并发下的库存扣减准确性）
- 优惠叠加规则（满减×优惠券×会员折扣）
- 退款金额计算（全额/部分、含运费与否）
- 秒杀/限时购的并发处理
- 多币种/多语言展示正确性

## 教育/在线教育

### 中国法规
| 法规名称 | 关键要求 | 测试关注点 |
|---------|---------|-------------|
| 《未成年人保护法》 | 未成年人防沉迷、内容适龄 | 实名认证、内容过滤 |
| 《关于进一步减轻义务教育阶段学生作业负担和校外培训负担的意见》（双减） | 学科类培训时间限制、预收费监管 | 排课、收费、退费 |
| 《教育移动互联网应用程序备案管理办法》 | 教育App备案 | App合规 |

### 教育行业特有测试关注点
- 课程内容的版权保护（水印、下载限制）
- 学习数据的隐私（谁可以查看学习记录）
- 防沉迷机制（未成年人使用时长限制、宵禁）
- 直播课的实时互动质量
- 题库/试卷的随机打乱与防作弊

|---------|---------|-------------|
| 《网络信息内容生态治理规定》 | 违法信息过滤、不良信息管理 | 内容审核、举报 |
| 《互联网信息服务管理办法》 | 信息服务的许可/备案 | 资质、内容管理 |
| 《网络音视频信息服务管理规定》 | 音视频内容审核 | 视频上传、直播 |
| 《互联网用户公众账号信息服务管理规定》 | 账号实名、运营规范 | 注册、认证、内容发布 |
| 《儿童个人信息网络保护规定》 | 14岁以下儿童信息处理需监护人同意 | 儿童用户注册、数据采集 |

### 通用
| 要求 | 说明 |
|------|------|
|  DSA（欧盟数字服务法） | 内容审核透明度、用户上诉机制 |
|  23条等平台责任法 | 用户生成内容管理、举报机制 |

### 社交/内容平台特有测试关注点
- 内容审核的覆盖率（敏感词、图片、视频）
- 举报处理流程的闭环
- 算法推荐的可解释性和关闭选项
- 用户可以下载自己的数据（数据可携带权）
- 账户删除/注销后的数据清理
- 未成年人的内容过滤

## 企业服务/SaaS

### 关注领域
| 要求 | 说明 | 测试关注点 |
|------|------|-------------|
| SLA 服务等级协议 | 可用性承诺（如99.9%）、故障响应时间 | 监控、告警、容灾 |
| SOC 2 | 安全性、可用性、保密性、隐私 | 访问控制、加密、审计日志 |
| ISO 27001 | 信息安全管理体系 | 数据保护、安全策略 |
| 《网络安全等级保护条例》（等保2.0） | 系统安全等级定级、保护措施 | 身份认证、访问控制、日志 |

### SaaS 特有测试关注点
- 多租户数据隔离（A公司用户不能看到B公司数据）
- 租户管理（创建、禁用、删除租户）
- 角色/权限体系（RBAC模型）
- 审计日志完整性
- 数据导出/导入（客户数据迁移场景）
- 资源配额（存储空间、API调用次数等限制）

## 常见合规测试点示例

### 用户数据相关（跨行业通用）

| 测试点 | 涉及法规 | 适用行业 |
|--------|---------|---------|
| 验证用户注册时需明确同意隐私政策 | 个人信息保护法、GDPR | 全行业 |
| 验证未勾选隐私政策时无法完成注册 | 个人信息保护法 | 全行业 |
| 验证用户可以导出其全部个人数据 | 个人信息保护法、GDPR | 全行业 |
| 验证用户注销账户后其数据在规定时间内被清除 | 个人信息保护法 | 全行业 |
| 验证未成年人注册需监护人授权 | 未成年人保护法、儿童隐私 | 社交、教育、游戏 |
| 验证密码以加密形式存储（不可明文查看） | 网络安全法、等保 | 全行业 |

### 业务合规相关

| 测试点 | 涉及法规 | 适用行业 |
|--------|---------|---------|
| 验证支付金额不超过对应账户等级限额 | 非银行支付机构管理办法 | 金融、支付 |
| 验证商品展示价格与实际结算价格一致 | 价格法、消费者权益法 | 电商 |
| 验证7天无理由退货流程可用 | 消费者权益保护法 | 电商 |
| 验证药品购买需输入处方信息 | 药品管理法 | 医疗、医药电商 |
| 验证反洗钱大额交易触发上报流程 | 反洗钱法 | 金融 |
| 验证学科类培训在法定时间外不可排课 | 双减政策 | 教育 |
| 验证UGC内容发布前经过敏感词审核 | 网络信息内容生态治理规定 | 社交、内容平台 |
| 验证多租户数据严格隔离 | 等保、SOC2 | SaaS |

## 技术新规：CMP（合规）

### 定义
CMP = Compliance（合规），作为第5种测试设计技术。

### 适用场景
当产品涉及以下领域时，必须生成 CMP 测试点：
1. 处理用户个人信息
2. 涉及金融交易或资金流转
3. 面向未成年人
4. 用户生成内容（UGC）
5. 涉及医疗健康信息
6. 公开/准公开的信息发布
7. 跨业务线/跨租户的数据流通

### CMP 测试点生成规则
1. 首先识别产品所属行业
2. 查找本文件中该行业的适用法规
3. 对每条法规提取可测试的合规要求
4. 将法规要求转化为测试点
5. 标注引用的法规名称

### CMP 测试点 ID 格式

格式：`[MODULE]-CMP-[SEQ]`

从ID可直接识别所属模块。

例如：
- AUTH-CMP-001：[P0] 验证隐私政策同意为注册必选项（个人信息保护法）
- PYMT-CMP-001：[P0] 验证支付限额符合Ⅰ/Ⅱ/Ⅲ类账户规定（支付机构管理办法）
