# 边界值分析（Boundary Value Analysis）

## 适用场景

有明确数值范围、长度限制或时间阈值的输入。

## 规则

- 闭区间 [A, B]：测试 A-1, A, A+1, B-1, B, B+1
- 开区间 (A, B)：测试 A, A+1, B-1, B
- 2值法：测试边界值和边界值相邻的值（min, min+1, max-1, max 及越界）
- 3值法：测试边界值和边界两侧的值（min-1, min, min+1, max-1, max, max+1）

## 常见场景

| 场景 | 边界值 | 测试点示例 |
|------|--------|-----------|
| 密码长度 [8,64] | 7,8,9,63,64,65 | "验证密码7/8/9/63/64/65字符的边界行为" |
| 金额 [0.01, 99999.99] | 0,0.01,0.02,99999.98,99999.99,100000 | "验证金额0/0.01/0.02边界值" |
| 超时时间 [1, 30] 分钟 | 0,1,2,29,30,31 | "验证会话超时29/30/31分钟的边界行为" |
| 列表最大50条 | 49,50,51 | "验证列表49/50/51条记录的边界行为" |

## 测试点编写规则

- ✅ "验证密码长度8-64字符边界值（7、8、9、63、64、65字符）"
- ✅ "验证单笔交易金额边界（0.01、99999.99及越界值）"
- ✅ "验证搜索返回结果50条上限（49、50、51条结果）"
- ❌ "在输入框输入7个字符"（含步骤）

## 注意事项

- 边界值测试通常与等价类划分配合使用
- 对于时间相关的边界，需要注意时区、闰秒等
- 当有多个输入字段都有边界时，优先测试关键字段的边界
