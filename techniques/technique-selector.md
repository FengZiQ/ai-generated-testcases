# 测试技术选择器

将需求映射到适用的测试设计技术。

## 映射规则

| 需求特征 | 主要技术 | 次要技术 | 示例关键词 |
|---------|---------|---------|-----------|
| 有效/无效输入分类 | 等价类划分 | — | valid, invalid, 有效, 无效, 必须, 允许 |
| 数值范围/限制 | 边界值分析 | 等价类划分 | between, range, 之间, 范围, 不超过, 至少, 最多, up to, max, min |
| 多条件组合（布尔） | 决策表 | — | if...then, when, unless, 如果, 当, 否则, 取决于 |
| 状态/状态变更 | 状态转换 | — | state, status, 状态, 阶段, 转换, 变更, transition, workflow |
| 错误/异常处理 | 等价类划分 | 状态转换 | error, timeout, invalid, 错误, 超时, 无效 |
| 具体数值约束 | 边界值分析 | — | 见到具体数字和限制词（8-64字符, 30分钟等） |
| 用户认证/权限 | **安全测试** | — | password, login, auth, permission, role, 权限, 角色, 认证 |
| 数据输入展示 | **安全测试** | — | input, upload, search, comment, 输入, 上传, 搜索, 评论 |
| 列表/搜索结果 | **性能测试** | — | list, search, report, 列表, 搜索, 报表, 导出 |
| 高并发业务 | **性能测试** | — | panic buying, flash sale, 秒杀, 抢购, 高并发, real-time |
| 复杂业务流程/未知风险 | **探索式测试** | — | charter, session, 探索, 不确定, 模糊, 复杂流程, 历史缺陷 |

## 选择流程

1. 逐个需求分析关键词模式
2. 匹配上表确定主要和次要技术
3. 对于决策表：列出所有条件（布尔条件）
4. 对于状态转换：列出所有状态和事件
5. 对于探索式测试：参考 `techniques/exploratory-testing.md` 设计 Session
6. 输出格式：

| 需求ID | 主要技术 | 次要技术 | 说明 |
|--------|---------|---------|------|
| R-NNN | BVA | EQP | 密码8-64字符边界 |
| R-NNN | DCS | — | 4个条件的登录决策 |

## 粒度选择指南

| 技术类型 | 推荐粒度 | 说明 |
|---------|---------|------|
| EQP 等价类划分 | L2（场景级） | 每个有效/无效类是一个场景，不应细分到单值 |
| BVA 边界值分析 | L3（条件级） | 边界值天生需要具体数值验证 |
| DCS 决策表 | L2（场景级） | 一行决策规则是一个场景 |
| STT 状态转换 | L2（场景级） | 一个状态转换路径是一个场景 |
| SEC 安全测试 | L2（场景级）或L1（功能级） | 单攻击类型=场景级；全量扫描=功能级 |
| PERF 性能测试 | L2（场景级） | 一个性能场景 |
| CMP 合规测试 | L2（场景级） | 一条法规的一个可测试点 |
| GEN 通用功能 | L1（功能级） | 简单功能点确认 |
| ET 探索式测试 | 不适用 | ET按Session组织，不适用测试点粒度分级 |

**选择原则**：如果使用EQP/BVA/DCS/STT等技术生成了多个测试点，检查是否都是同一粒度级别。如果某功能的测试点出现"1个L1+若干L3"，说明粒度层级用混了，需要将L1拆为L2或将L3合并为L2。

## 探索式测试选择指南

当需求/变更具有以下特征时，建议分配探索式测试资源：

1. 涉及3个以上模块交互的变更
2. 变更影响分析为"高"的模块
3. 历史缺陷数量超过平均值的模块
4. 外部系统集成的新增或修改
5. 需求描述模糊或存在多个"待确认"标注的领域

## 常用映射速查

- **表单验证** → EQP（各类输入）+ BVA（数值字段）
- **业务规则** → DCS（条件组合）
- **审批流/订单流** → STT（状态转换）
- **搜索/过滤** → EQP + BVA
- **权限控制** → DCS（角色×资源×操作）+ **SEC（越权测试）**
- **购物车/订单** → STT（状态机）+ DCS（条件规则）
- **所有模块** → SEC（安全测试：XSS/注入/越权/敏感数据暴露）
- **所有产品** → PERF（性能测试：响应时间/并发/稳定性）
- **搜索/列表/导出** → PERF（性能测试：大数据量下的响应时间）
- **登录/支付/订单** → SEC（安全测试：重点覆盖）
