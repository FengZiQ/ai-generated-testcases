# 性能测试（Performance Testing）

## 适用场景

所有产品，特别是在以下方面有明确要求或高预期的产品：
- 高并发用户访问
- 大量数据处理
- 实时/近实时响应
- 资源消耗敏感环境（移动端、嵌入式）

## 性能测试分类及测试点生成指南

### 1. 响应时间

| 测试领域 | 阈值参考 | 测试点示例 |
|---------|---------|-----------|
| 页面加载 | 首屏<3s，完全加载<5s | "验证页面首屏加载时间不超过3秒" |
| API响应 | <500ms（简单查询），<2s（复杂操作） | "验证核心API（登录、搜索）响应时间不超过500ms" |
| 文件上传/下载 | 进度反馈 | "验证大文件上传（>100MB）有进度提示且不超时" |
| 列表查询 | 分页加载<1s | "验证万级数据列表分页查询响应时间不超过1秒" |
| 搜索响应 | <3s | "验证复杂条件搜索在3秒内返回结果" |

### 2. 并发处理

| 测试领域 | 测试点示例 |
|---------|-----------|
| 多用户并发 | "验证100用户同时登录系统不崩溃" |
| 热点数据 | "验证秒杀/抢购场景下500人同时请求不超卖" |
| 资源竞争 | "验证多个线程同时写入同一数据时无死锁" |
| 连接池 | "验证数据库连接池耗尽时优雅降级而非崩溃" |
| 队列处理 | "验证高并发下的异步任务不被重复处理" |

### 3. 负载与容量

| 测试领域 | 测试点示例 |
|---------|-----------|
| 常规负载 | "验证200用户同时在线的系统运行稳定" |
| 峰值负载 | "验证双11/促销场景下平时10倍流量的系统表现" |
| 数据容量 | "验证千万级数据量下的查询和导出性能" |
| 存储极限 | "验证数据库达到设计容量上限时的行为" |
| 缓存效果 | "验证热点数据缓存命中率达标（>90%）" |

### 4. 稳定性与长时间运行

| 测试领域 | 测试点示例 |
|---------|-----------|
| 长时间运行 | "验证系统72小时持续运行无内存泄漏" |
| 内存管理 | "验证长时间操作后内存不持续增长（无泄漏）" |
| 定时任务 | "验证日切/对账等定时任务在规定时间内完成" |
| 日志轮转 | "验证大量日志写入不影响系统性能" |

### 5. 资源利用率

| 测试领域 | 阈值参考 | 测试点示例 |
|---------|---------|-----------|
| CPU | <70% | "验证典型负载下CPU使用率不超过70%" |
| 内存 | <80% | "验证典型负载下内存使用率不超过80%" |
| 磁盘I/O | 等待时间<100ms | "验证高并发写入时磁盘I/O不成为瓶颈" |
| 网络带宽 | <60% | "验证文件批量导出时带宽使用合理" |
| 移动端耗电 | — | "验证后台定位/推送功能不过度消耗电量" |

### 6. 可扩展性

| 测试领域 | 测试点示例 |
|---------|-----------|
| 水平扩展 | "验证增加服务器节点后吞吐量线性提升" |
| 数据库扩展 | "验证读写分离/分库分表后的查询性能" |
| 弹性伸缩 | "验证云环境下自动扩缩容对性能的影响" |

## 非功能需求推断规则

当需求文档没有明确性能指标时，根据产品类型推断合理阈值：

| 产品类型 | 页面加载 | API响应 | 并发用户参考 |
|---------|---------|---------|------------|
| B端管理后台 | <5s | <2s | 50-500 |
| C端电商/社交 | <3s | <500ms | 1000-10000 |
| 金融交易系统 | <2s | <200ms | 500-5000 |
| 实时协作工具 | <1s | <200ms | 200-2000 |
| 移动端App | <2s | <1s | — |
| API/中间件 | — | <100ms | 1000-100000 |

## 性能测试点生成规则

1. **基于产品类型推断阈值**：使用上表中的产品类型参考阈值
2. **识别性能敏感功能**：列表查询、搜索、报表导出、文件上传下载、实时推送等
3. **区分场景等级**：
   - 核心场景（P0）：用户主流程涉及的功能
   - 重要场景（P1）：高频使用的功能
   - 一般场景（P2）：低频或后台功能
4. **标注预期阈值**：每个性能测试点标明具体的性能预期值

## 性能测试点ID格式

格式：`[MODULE]-PERF-[SEQ]`

- 跨模块的性能测试点，MODULE 使用 SYS
- 从ID可直接识别所属模块
- 示例：
  - SYS-PERF-001：[P0] 验证页面首屏加载时间不超过3秒
  - ORDR-PERF-001：[P1] 验证万级订单列表查询不超过1秒
  - SRCH-PERF-001：[P1] 验证搜索响应时间不超过3秒

## 性能测试点编写规则

- ✅ "验证100用户并发登录时响应时间不超过2秒"
- ✅ "验证千万级数据量下的模糊搜索在5秒内返回"
- ✅ "验证报表导出功能在10万条数据时不超过30秒"
- ✅ "验证72小时持续运行后内存使用率不增长超过5%"
- ❌ "测试性能"（太笼统）
- ❌ "点击搜索→查看搜索结果多久出来"（含步骤）
